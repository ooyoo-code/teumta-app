# ⚡ 틈타 (Teumta) - AI 초단기 일자리 매칭 플랫폼 MVP

본 프로젝트는 일본의 검증된 '스키마 바이토(틈새 시간 초단기 알바)' 모델을 한국 시장에 적용하기 위한 MVP(Minimum Viable Product, 최소 기능 제품) 프로토타입입니다.

개인의 **남는 시간/위치**와 사장님의 **갑작스러운 구인 요청**을 면접 없이 1초 만에 실시간으로 매칭하는 핵심 사용자 흐름(Core Loop)을 완벽히 시뮬레이션할 수 있습니다.

## 📂 프로젝트 구조

실제 출시 형태(구직자 앱 / 사장님 앱 분리)와 동일하게 두 개의 독립된 앱으로 구성되어 있습니다.

```
teumta-app/
├── index.html        # 앱 선택(구직자/사장님) 랜딩 페이지
├── seeker/           # 구직자 앱
├── employer/         # 사장님 앱
└── shared/           # 두 앱이 공유하는 스타일 · 매칭 엔진 · 데이터 레이어
    ├── style.css
    ├── matching.js   # 시간/거리(하버사인 공식)/업종 매칭 로직
    └── db.js         # 데이터 레이어 (현재 localStorage, Firestore로 교체 예정)
```

`shared/db.js`는 오늘은 `localStorage` + 브라우저 `storage` 이벤트로 두 앱이 같은 브라우저의 서로 다른 탭에서 실시간으로 동기화되도록 만들어졌고, 추후 Firebase(Firestore)로 교체해도 `seeker/app.js`·`employer/app.js`는 거의 수정할 필요가 없도록 설계했습니다.

## 🚀 주요 제공 기능

* **구직자 모드 ⚡**
  * **내 남는 시간 및 활동 희망 지역 등록**: 달력과 범위 설정을 통해 나만의 근무 가능 윈도우 지정.
  * **1초 매칭 시스템**: 내 시간/위치 조건과 맞는 공고가 올라올 시, 면접 없이 즉시 매칭 수락.
  * **출퇴근 체크**: 근무 시작 및 완료 체크를 통해 사장님에게 근무 상태 실시간 전달.
  * **간편 정산 지갑**: 근무 완료 및 승인 후 즉시 적립되는 가상 지갑 및 누적 급여 현황판.

* **사장님 모드 💼**
  * **3초 긴급 구인 양식**: 모집 분야, 시급, 근무 시간대, 근무지를 입력해 즉시 일거리 생성.
  * **실시간 매칭 상태 관리**: 등록한 구인 공고에 구직자가 매칭되었는지 확인하고, 근무자의 실시간 상태 모니터링.
  * **급여 지급 승인**: 근무가 완료된 구직자에게 원클릭으로 급여를 송금하는 정산 승인 프로세스.

* **MVP 매칭 시뮬레이터 🛠️**
  * 테스트 편의성을 위한 샘플 구인 공고 자동 생성 및 데이터 초기화 컨트롤러 제공.
  * `localStorage`를 활용하여 새로고침을 하거나 모드를 넘나들어도 데이터 상태가 실시간으로 동기화되고 유지됨.

본 애플리케이션은 서버가 필요 없는 정적 웹 앱(HTML/CSS/JS)입니다.

## 🌐 GitHub Pages를 통해 무료로 서비스 배포하기

이 MVP 프로젝트를 GitHub에 올려 웹 서비스 형태로 호스팅(배포)하려면 다음 단계를 따르세요.

### 1단계: Git 저장소 초기화 및 첫 커밋
프로젝트 디렉토리에서 터미널을 열고 아래 명령어를 입력합니다.
```bash
git init
git add .
git commit -m "feat: 초단기 알바 매칭 플랫폼 MVP 완성"
```

### 2단계: GitHub 원격 리포지토리 연결 및 푸시
GitHub 사이트에서 새로운 빈 리포지토리(예: `sukima-baito-mvp`)를 생성한 후, 아래 명령어를 실행하여 로컬 저장소와 연결하고 푸시합니다. (사용자명과 리포지토리 이름은 본인 계정에 맞게 변경하세요)
```bash
git branch -M main
git remote add origin https://github.com/사용자아이디/리포지토리이름.git
git push -u origin main
```

### 3단계: GitHub Pages 서비스 활성화
1. GitHub 리포지토리 페이지로 이동하여 우측 상단의 **Settings** 메뉴를 클릭합니다.
2. 왼쪽 사이드바 메뉴에서 **Pages**를 클릭합니다.
3. Build and deployment의 **Source**가 `Deploy from a branch`로 설정되어 있는지 확인합니다.
4. **Branch** 설정에서 `main` 브랜치와 `/ (root)` 폴더를 선택한 뒤 **Save** 버튼을 누릅니다.
5. 약 1~2분 후 페이지 상단에 생성된 배포 URL(예: `https://username.github.io/repository-name/`)이 나타납니다. 이 링크를 통해 스마트폰이나 웹 브라우저에서 즉시 접속하여 서비스를 이용하실 수 있습니다!
